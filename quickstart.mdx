This guide will help you set up and run TaskFlow Angular, a Kanban-style project management application built with Angular 21.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 18.19 or later
- **npm** 11.6.1 or later (automatically installed with Node.js)
- **Backend API** running on `https://localhost:7057` (ASP.NET Core with MySQL)

<Note>
TaskFlow Angular requires a backend API to function. Ensure the backend is configured and running before starting the frontend application.
</Note>

<Steps>

<Step title="Clone and navigate to the project">

Clone the repository and navigate to the project directory:

```bash
git clone <repository-url>
cd task-flow-angular
```

</Step>

<Step title="Install dependencies">

Install all required packages using npm:

```bash
npm install
```

This will install:
- Angular 21.1.0 core packages
- Angular CDK 21.1.5 for drag-and-drop functionality
- RxJS 7.8.0 for reactive programming
- Development tools including TypeScript 5.9.2 and Vitest 4.0.8

</Step>

<Step title="Start the development server">

Run the application in development mode:

```bash
npm start
```

Or use the Angular CLI directly:

```bash
ng serve
```

The application will be available at `http://localhost:4200`

</Step>

<Step title="Access the application">

Open your browser and navigate to:

```
http://localhost:4200
```

You'll be redirected to the login page. Create a new account using the register link or log in with existing credentials.

</Step>

</Steps>

## Available scripts

TaskFlow Angular includes several npm scripts for different tasks:

<CodeGroup>

```bash Development
npm start
# Starts the development server with hot reload
```

```bash Build
npm run build
# Creates an optimized production build
```

```bash Watch Mode
npm run watch
# Builds the project and watches for changes
```

```bash Tests
npm test
# Runs the test suite using Vitest
```

</CodeGroup>

## Project structure

Once the application is running, you'll find the following structure:

```
src/app/
├── components/          # Reusable UI components
│   ├── layout/         # Header, sidebar, and layout components
│   ├── column/         # Kanban column components
│   ├── task/           # Task card and modal components
│   └── ui/             # Generic UI components (modals, etc.)
├── pages/              # Route-based page components
│   ├── auth/           # Login and register pages
│   └── boards/         # Board list and detail pages
├── services/           # Business logic and API communication
├── guards/             # Route protection (authGuard)
├── interceptors/       # HTTP interceptors (JWT authentication)
└── models/             # TypeScript interfaces and types
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Application won't start">
    Ensure you have the correct Node.js version installed (18.19 or later). Check that all dependencies installed successfully by running `npm install` again.
  </Accordion>
  
  <Accordion title="Cannot connect to backend API">
    Verify the backend API is running on `https://localhost:7057`. Check the `apiUrl` configuration in your service files (e.g., `src/app/services/auth.service.ts:11`).
  </Accordion>
  
  <Accordion title="Port 4200 is already in use">
    Kill the process using port 4200 or specify a different port: `ng serve --port 4300`
  </Accordion>
</AccordionGroup>

<Warning>
The default API URL is configured as `https://localhost:7057`. Make sure to update this in your service files if your backend runs on a different URL.
</Warning>
