## Overview

The `ConfirmModal` component provides a simple confirmation dialog that displays a message with confirm and cancel buttons. It's useful for prompting users before destructive or important actions.

## Component Definition

```typescript
import { Component, Input, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-confirm-modal',
  standalone: true,
  templateUrl: './confirm-modal.html',
  styleUrl: './confirm-modal.css'
})
export class ConfirmModal {
  @Input() message = '¿Estás seguro?';
  @Output() confirm = new EventEmitter<void>();
  @Output() cancel = new EventEmitter<void>();

  onConfirm() {
    this.confirm.emit();
  }

  onCancel() {
    this.cancel.emit();
  }
}
```

## Inputs

<ParamField path="message" type="string" default="¿Estás seguro?">
  The confirmation message to display in the modal
</ParamField>

## Outputs

<ParamField path="confirm" type="EventEmitter<void>">
  Emitted when the user clicks the confirm button
</ParamField>

<ParamField path="cancel" type="EventEmitter<void>">
  Emitted when the user clicks the cancel button
</ParamField>

## Usage

### Basic Example

```typescript
import { Component } from '@angular/core';
import { ConfirmModal } from './components/ui/confirm-modal/confirm-modal';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [ConfirmModal],
  template: `
    <app-confirm-modal
      message="¿Estás seguro de que quieres eliminar este elemento?"
      (confirm)="handleConfirm()"
      (cancel)="handleCancel()">
    </app-confirm-modal>
  `
})
export class AppComponent {
  handleConfirm() {
    console.log('User confirmed the action');
    // Perform the confirmed action
  }

  handleCancel() {
    console.log('User cancelled the action');
    // Close the modal or revert
  }
}
```

### Custom Message Example

```typescript
@Component({
  selector: 'app-delete-task',
  template: `
    <app-confirm-modal
      [message]="confirmMessage"
      (confirm)="deleteTask()"
      (cancel)="closeModal()">
    </app-confirm-modal>
  `
})
export class DeleteTaskComponent {
  confirmMessage = '¿Eliminar esta tarea permanentemente?';

  deleteTask() {
    // Delete the task
    this.closeModal();
  }

  closeModal() {
    // Close the modal
  }
}
```

## Template Structure

The component uses the following HTML structure:

```html
<div class="modal-backdrop">
  <div class="confirm-modal">
    <p>{{ message }}</p>
    <div class="actions">
      <button class="cancel-btn" (click)="onCancel()">
        Cancelar
      </button>
      <button class="confirm-btn" (click)="onConfirm()">
        Confirmar
      </button>
    </div>
  </div>
</div>
```

## Styling

The component includes the following CSS classes that can be customized:

- `.modal-backdrop` - The semi-transparent overlay behind the modal
- `.confirm-modal` - The main modal container
- `.actions` - Container for the action buttons
- `.cancel-btn` - The cancel button
- `.confirm-btn` - The confirm button

<Note>
  This is a standalone component and can be imported directly without a module.
</Note>

## Best Practices

<Tip>
  Always provide clear, specific messages that explain what action is being confirmed to avoid user confusion.
</Tip>

<Warning>
  Remember to handle both the confirm and cancel events to provide a complete user experience.
</Warning>
